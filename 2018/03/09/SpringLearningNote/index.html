Spring学习笔记

记笔记的确是个好习惯，能够把学习过程中自己的想法和思考记录下来。
所以，这回学习Spring的时候我就边学编写了这篇笔记，有些地方可能会有不足之处，还望指正！

关于使用框架

使用框架的基本思想是为了将程序的各个部分进行解耦，并解决或避免一些原生编程中的问题

IOC(控制反转)和DI(依赖注入)

配置文件方式

使用 <bean> 标签配置对象，在 <bean> 的内部使用 <property> 标签将数据注入对象，
对象的注入支持复杂对象（例如List、Map等）

注解方式

Spring支持使用注解方式来替代配置文件。首先需要在配置文件中写上
<context:component-scan base-package="被扫描的文件路径"/> 开启扫描。
有两种方式对对象进行创建

  • @Autowired
  • @Resource(value=”注入对象name”)
    等效于配置文件中的 <property name="xxx" ref="注入对象name">

混合使用

配置文件和注解方式是可以混合使用的

AOP(面向切面编程)

即不通过修改源代码实现功能扩展，横向机制，通过代理类实现

了解代理类之前的猜测理解：
通过Spring，创建需要被增强的类的子类，实现父类需要被增强的方法(即父类该方法中的所有功能)，
并加入新的增强内容

概念

个人理解：目标对象增强前可以理解为一块什么也没的奶油蛋糕，增强是各种口味的巧克力片
插入巧克力片的过程，就是织入(配置文件、注解)；插入巧克力片的位置，就是切入点；
在没有AOP的做法前，我们要强化一个蛋糕的口味，只能通过重新修改一个蛋糕。
而通过AOP的方式，我们只需要给蛋糕增强就可以了。

  • 连接点：类中可以被增强的方法称为连接点
  • Pointcut(切入点)： 在类中实际被增强的方法称为切入点(类中有多个可被增强的方法)
  • Advice(通知/增强)： 指增加（扩展）的功能
    • 前置通知： 在被增强方法执行之前执行
    • 后置通知： 在被增强方法执行之后执行
    • 异常通知： 在被增强方法执行之前执行
    • 最终通知： 在后置通知之后执行，有点类似finally的意思
    • 环绕通知： 在方法之前和之后执行
  • Aspect(切面)： 把增强应用到切入点的过程
  • 目标对象： 即需要被增强的对象
  • Weaving(织入)： 将对象增强功能应用到(织入)目标的过程

AspectJ

Spring中的AOP操作由AspectJ实现，有以下两种方案

注意在IDEA里建的Spring项目，是不会自动下载导入aspectjweaver-1.8.13.jar
这个依赖的，需要手动导入

优化

每次读取xml文件都需要花费较长时间new ClassPathXmlApplicationContext("xxx.xml");。
所以，通过服务器启动时加载配置文件的方式提升性能。
底层实现则是使用监听器和ServletContext对象，Spring已经封装好了监听器，只需要进行配置即可。
需要导入Spring-web.jar
在web.xml中进行配置：

<listener>
    <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>

同时此处一般要指定Spring配置文件的位置，否则将会是默认：

<context-param>
    <param-name>contextConfigLocation</param-name>
    <!--默认包根路径下-->
    <param-value>classpath:配置文件.xml</param-value>
</context-param>

表达式配置切入点 进行增强

表达式：execution(<访问修饰符>?<返回类型><方法名>(<参数>)<异常>)

示例：

  • execution(* com.narcissu14.testspring.print(..))

    指代指定路径包名下方法的，任意访问修饰符的方法
  • execution( com.narcissu14.testspring.(..))

    指代指定路径包名下，任意访问修饰符的所有方法
  • execution( \.*)

    匹配所有方法
  • execution( com.narcissu14.testspring.get(..))

    指代指定路径包名下名称以get为前缀的，任意访问修饰符的方法

步骤：

首先创建对象及其类(Java的代码部分此处不贴出了)

<bean id="aopBook" class="com.narcissu14.testspring.entity.AopBook"/>
<bean id="aopBookEnhance" class="com.narcissu14.testspring.entity.AopBookEnhance"/>

对切入点和增强进行配置

<!--配置AOP操作-->
<aop:config>
    <!--配置切入点-->
    <aop:pointcut expression="execution(* com.narcissu14.testspring.entity.AopBook.print())" id="aopBook-print"/>
    <!--配置切面，将增强织入-->
    <aop:aspect ref="aopBookEnhance">
        <aop:before method="enhanceBefore" pointcut-ref="aopBook-print"/>
        <aop:around method="enhanceAround" pointcut-ref="aopBook-print"/>
    </aop:aspect>
</aop:config>

注解设定切入点 进行增强

步骤：

  1. 首先创建bean配置，这个同上
  2. 增加<aop:aspectj-autoproxy/>开启自动代理
  3. 在增强类上方加上注解@Aspect
  4. 在增强方法上加通知类型的注解，有以下5种：
    • @Before：前置通知
    • @AfterReturning：后置通知
    • @Around：异常通知
    • @AfterThrowing：环绕通知
    • @After：最终通知
  5. 在通知类型之后加上表达式以指向需要被增强的类

    例如：
    @Before(value = “execution(* com.narcissu14.testspring.entity.AopBook.print())”)

JDBCTemplate

是对JDBC的封装，使用前需要先创建JdbcTemplate对象

//设置数据库信息
 DriverManagerDataSource dataSource = new DriverManagerDataSource();
 dataSource.setDriverClassName("com.mysql.jdbc.Driver");
 dataSource.setUrl("jdbc:mysql:///spring_test");
 dataSource.setUsername("root");
 dataSource.setPassword("");

 //创建JdbcTemplate对象，设置数据源
 JDBC_TEMPLATE = new JdbcTemplate(dataSource);

之后可以通过JDBC_TEMPLATE 来进行crud操作

关于查询

  • 查询并获取对象，可以使用JDBC_TEMPLATE.queryForObject(...)
    该方法返回一个指定类型的对象
    其中的”RowMapper”类型参数需要手动进行实现，主要实现

    1. 从结果集中得到数据
    2. 将获得的数据封装为对象后返回

Spring中配置C3P0连接池

实际上就是利用IOC，使用配置文件的方式创建并注入对象。
这里再次注意，默认情况下，创建对象是Singleton，即单例的。
例：

<!--配置C3P0连接池-->
 <bean id="c3p0" class="com.mchange.v2.c3p0.ComboPooledDataSource">
     <property name="driverClass" value="com.mysql.jdbc.Driver"/>
     <property name="jdbcUrl" value="jdbc:mysql:///spring_test"/>
     <property name="user" value="root"/>
     <property name="password" value=""/>
 </bean>

Spring的事务管理

  • Spring中事务的两种管理方式：
    • 编程式事务管理(一般不用)
    • 声明式事务管理
      • 基于xml配置文件实现
      • 基于注解方式实现
  • Spring事务管理的介绍
    • 事务管理器接口
      1. Spring对于不同的dao层框架，提供了接口不同的实现类
      2. 首先必须配置事务管理器

设计思想

  • Service层：业务逻辑层(对业务逻辑进行处理，具体数据crud交给dao)
  • Dao层：数据操作层(对数据进行crud)
